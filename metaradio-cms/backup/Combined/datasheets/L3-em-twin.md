# Challenge · 数字孪生缺失的一环

**Title:** 数字孪生为何缺一个"电磁层"
**Description:** 城市、园区、工厂、低空数字孪生项目层出不穷，但绝大多数停留在几何与物联感知层。一旦涉及无线网络规划、ISAC、低空 C2、卫星协同、智能驾驶通信，缺失"电磁层"就意味着孪生体无法支撑端到端通信测试与策略决策。

## Challenge Cards

| Icon | Title | Text |
| --- | --- | --- |
| 🧱 | 几何 ≠ 电磁 | 已有数字孪生只描述"看得见的"，对电磁波的反射、绕射、穿透与时变多径几乎不可见。 |
| 🔁 | 现实反馈断链 | 真实网络的覆盖、干扰、丢包数据无法回灌仿真，反向校正与优化无从谈起。 |
| 🏗 | 端到端缺基础 | 没有持续在线、可被设备访问的电磁环境，自动驾驶/低空/工业的端到端测试只能片段化。 |

# Solution · 解决方案

**Title:** 把电磁世界变成"持续在线"的孪生平台
**Description:** EM-Twin 平台以 Lauraycs 射线跟踪为电磁内核，叠加来自现网/工厂/低空/星地的实时数据，并通过 HIL 子系统对真实终端持续可见。它不是一次性仿真，而是一个能被订阅、被查询、被驱动的电磁环境服务。

# Architecture · 平台架构

**Title:** 电磁孪生的四层架构
**Description:** 从底层环境数据采集，到电磁内核计算，到对外服务接口，再到与上层数字孪生的对接，构成完整电磁孪生平台。每一层均提供标准 API。

## Workflow

| Step | Title | Desc |
| --- | --- | --- |
| 01 | 多源环境数据 | OSM / BIM / 倾斜摄影 / 激光点云 / 实时 IoT；与城市/工厂数字孪生底图对齐。 |
| 02 | RT 电磁内核 | Lauraycs 射线跟踪持续运算时变多径与覆盖；GPU 集群弹性扩展。 |
| 03 | 服务化 API | gRPC / REST / Streaming；订阅指定地理盒子或时间窗口的电磁状态。 |
| 04 | 上层应用与回灌 | 网络规划、ISAC、低空、自动驾驶、HIL、AI 训练；现实数据回灌持续校正。 |

# Differentiators · 核心差异化

## Bullets

- **电磁层填空：** 真正"补齐"数字孪生的电磁维度，而非把覆盖图静态贴图。
- **可被驱动：** 不只是看，更可作为信道源驱动 HIL/OTA 真实硬件测试。
- **现实闭环：** 实测数据持续回灌，孪生保真度随时间收敛而非衰减。
- **可订阅的电磁服务：** 允许第三方应用通过 API 订阅电磁状态，孪生作为公共基础设施。
- **跨域底图：** 城市、园区、工厂、低空、星地多种孪生体系可统一在一个电磁内核上。
- **AI 数据工厂：** 输出大规模标注数据集，支撑 ISAC / 6G AI-native channel 与定位模型训练。
